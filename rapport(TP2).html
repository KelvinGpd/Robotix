Analyse

Plusieurs données doivent être informatisées, notamment le compte de chaque utilisateur.
Pour un utilisateur régulier: nom, addresse courielle, robots associés, etc. Un fournisseur est un utilisateur
sauf qu'en plus il y a les composantes associées et la capacité de fabrication. Le système de points
requiert également le stockage des points. Pour chaque robot, il faut informatiser son uuid, son type
ainsi que toutes les informations reliées à l'éxécution des tâches (dates, métriques...).
Pour les composantes: identifiant, description et prix. Le processus de collection et écriture
sera informatisé(Traitement batch) et les données seront stockées sur une base de données.
Matériel: Sauvegardes - Mémoire de masse(emprunté à aws ou autre...)
Le système Robotix interagit alors en read/write avec le système de stockage de données.
Les utilisateurs interagissent avec Robotix en créeant des objets
Robot, activités, compte utilisateur, etc. avec les attributs nécessaires en utilisant le système.
